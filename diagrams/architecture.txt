Data Flow:
  1. Client sends request to llama-server at max1:8080 (OpenAI-compatible API)
  2. llama-server splits model layers across 2 RPC backends (-ts 1,1)
  3. Each rpc-server uses local Vulkan GPU to compute its share
  4. Tensor data flows between nodes over USB4 v2 Thunderbolt (TCP/RPC)
  5. llama-server assembles results and streams response to client

Key Configuration:
  - GGML_VK_VISIBLE_DEVICES="" on head node (prevents double-counting GPU)
  - --no-mmap required for distributed inference
  - -ts 1,1 splits layers evenly across both GPUs
  - Flash attention (-fa 1) and KV cache quantization (-ctk q8_0) save VRAM

Model: MiniMax-M2.5 229B (UD-Q6_K_XL, ~181 GB)
  - 229B total params, 10B active per token
  - 32 experts, 2 active per token, 61 layers
  - ~90 GB per node after KV cache allocation
  - Generation: ~13 tok/s | Prompt: ~195 tok/s
